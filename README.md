# gpu-attention-kernels

A CUDA/C++ implementation of naive scaled dot-product attention (forward pass), built as a performance baseline for comparing against tiled, fused, and FlashAttention-style optimizations.

This is a **systems/performance project, not an ML project** — no backward pass, no training, no model quality. The only goal is to measure runtime and memory usage of a realistic attention workload and to serve as an honest baseline for future optimization work.

## What's implemented

Three GPU implementations, plus a CPU reference for correctness:

| Implementation | What it is |
|---|---|
| **naive** | cuBLAS for projections + Q·Kᵀ + P·V; custom kernels only for softmax and head transposes. Materializes the full `[B, H, N, N]` attention matrix in HBM. The standard "by-the-book" baseline. |
| **flash (simple CUDA)** | Hand-written fused kernel with online softmax and tiling. Never materializes S. Demonstrates the algorithmic structure of FlashAttention without the kernel-level engineering — uses per-thread scalar arithmetic instead of warp-cooperative tensor cores. |
| **flash (official)** | Real FlashAttention-2, accessed via PyTorch's `scaled_dot_product_attention` with the flash backend forced. The production reference. |

The third implementation is benchmarked from a Python script that loads the same `.npy` input tensors as the C++ benchmarks, so all three see identical bytes.

## Project structure

```
gpu-attention-kernels/
├── CMakeLists.txt
├── main.cu                       # benchmark driver (C++ side)
├── bench_flash_attn.py           # benchmark driver (Python side, flash-attn)
├── requirements.txt              # Python dependencies
├── README.md
├── benchmarks/                   # benchmark utilities
├── data/                         # generated .npy input tensors
├── include/                      # public headers (if any)
├── python/                       # data generation scripts
├── scripts/                      # shell drivers (run_all_benchmarks.sh, etc.)
├── src/
│   ├── kernels/
│   │   ├── common.cuh            # CUDA error macros, GPU timer, mem helpers
│   │   ├── kernels.cuh           # kernel declarations
│   │   ├── kernels.cu            # softmax + transpose kernels
│   │   ├── tiled.cu              # tiled GEMM kernels (work in progress)
│   │   ├── online_softmax.cu     # online-softmax device functions
│   │   └── fused.cu              # fused attention kernel (online softmax + tiling)
│   ├── utils/
│   │   └── npy_loader.h          # load .npy files from C++
│   ├── wrappers/
│   │   ├── attention.h           # public API + workspace struct
│   │   ├── attention_naive.cu    # cuBLAS-based naive forward pass
│   │   ├── attention_tiled.cu    # tiled forward pass
│   │   ├── attention_fused.cu    # fused forward pass
│   │   ├── attention_flash.h     # flash-style header
│   │   └── attention_flash.cu    # flash-style fused kernel orchestration
│   └── reference.cpp             # CPU reference (correctness)
└── build/                        # generated by CMake
```

## Build

Requirements: CUDA 11+, CMake 3.18+, a C++17 compiler.

```bash
mkdir -p build
cd build
cmake ..
make -j
```

---

## Data Generation

We use fixed input data to ensure reproducible benchmarking across runs.

Generate data:

```bash
cd python
python3 generate_data.py
```

This creates:

```
data/
  X_128.npy
  Wq_128.npy
  Wk_128.npy
  Wv_128.npy
  ...
```

Each `.npy` carries its own dtype and shape header, so the loaders in both C++ (`src/utils/npy_loader.h`) and Python (`np.load`) read them without out-of-band metadata.

### Run the Python flash-attn benchmark

**One-time setup on Perlmutter (or similar HPC environments).** Python and `pip` aren't on the default `PATH` — you have to load the `pytorch` module first, which provides a Python environment with PyTorch and CUDA already configured:

```bash
module load pytorch
pip install --user flash_attn
```

The `--user` flag installs into `~/.local/`, which persists across sessions and across nodes. You only need to do this once. After that, every future session just needs `module load pytorch` and `flash_attn` will be importable.


Verify the install worked:

```bash
python -c "import flash_attn; print(flash_attn.__version__)"
```

Then, on a GPU node with `data/` already populated:

```bash
python bench_flash_attn.py --tensor-dir data
```

This loads the `.npy` files from `data/`, runs the FlashAttention-2 kernel, and emits CSV in the same format as the C++ benchmarks.

### Run everything together

```bash
bash scripts/run_all_benchmarks.sh
```

This runs the C++ benchmarks (naive, tiled, fused) then the Python flash-attn benchmark, and concatenates results into a single CSV.

### Metrics

- **mean_ms / min_ms / max_ms** → runtime statistics  
- **workspace_MB** → memory used by intermediate buffers  
- **peak_MB** → peak GPU memory usage  
- **max_abs_err** → correctness vs CPU reference  
  - `skip` = correctness not checked (large inputs)

## Correctness Checking

For small sequence lengths (≤ 512), we compare:

```
GPU implementation vs CPU reference
```

using max absolute error.

Large inputs skip correctness due to high CPU cost.

## Adding New Implementations

### Step 1: Add wrapper

Create a new file:

```
src/wrappers/attention_<name>.cu
```

Implement:

```cpp
void attention_forward_<name>(...);
```

---

### Step 2: Register in `attention.h`

```cpp
void attention_forward_<name>(...);
```

---

### Step 3: Add to benchmark

Update kernel selection in `benchmark.cu`:

```cpp
if (kernel == "<name>") {
    attention_fn = attention_forward_<name>;
}
```

---

### Step 4: Add to CMake

```cmake
src/wrappers/attention_<name>.cu
```

---

### Step 5: Rebuild

```bash
cd build
rm -rf *
cmake ..
make -j
```


## Configuration

Defaults in `main.cu` (and matching `python/generate_data.py`):

```cpp
batch    = 1
n_heads  = 8
d_head   = 64
seq_lens = {128, 256, 512, 1024, 2048, 4096, 8192}
warmup   = 3
iters    = 10
```

These are sized so the largest S tensor (~2 GB at `seq=8192`) fits comfortably on a 16 GB+ GPU. If you have less memory, drop `seq=8192` or reduce `n_heads`. If you have more, add `16384` — but note S is then ~17 GB.
